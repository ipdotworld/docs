## Trend fields

### Image

Primary card art for the umbrella page.

### Canonical name and aliases

**Name** is canonical; **aliases** catch alternate spellings and search variants.

<Tip>
  Add aliases the way people actually type the meme.
</Tip>

### Ownership type

- **Creator trend** — optional owner link for verification.
- **Community trend** — token-holder coordinated; token launch required.

See [Creator vs community trends](/user-guides/verification/creator-vs-community-trends).

### Category

Categories organize Explore. Current examples include **Pet**, **Person**, **Trend**, **Technology**, **Meme**, **Agent**, **Movement**, **Brand**, **Art**, **Misc**—follow the live picker.

### Trend socials

Website, **X**, **TikTok**, **Instagram**, **YouTube** when exposed in the form.

## Token fields

### Token image, name, ticker

**Ticker** is shown with **`$`** prefix in feeds and search.

### Token socials

Adds **Telegram** in addition to the trend social list.

## Copy trend detail to token

When launching the first token with a new trend, you can copy image/description/socials as a starting point—still customize when the market needs different positioning.

## Anti-snipe

When enabled at launch, the product enforces short post-launch **per-wallet supply caps** (example shipped in older copy: **1% of supply for 5 minutes**). **Always read the create screen**—limits can change.

Trades that would violate limits surface errors such as **Anti-snipe limit exceeded**.

## Creator buy

Buy amount is denominated in **`IP`** with UI estimates for tokens received, % supply, and max purchase limits.
