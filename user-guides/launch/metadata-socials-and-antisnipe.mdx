## Trend fields

### Image

Primary card art for the umbrella page.

### Canonical name and aliases

**Name** is canonical; **aliases** catch alternate spellings and search variants.

<Tip>
  Add aliases the way people actually type the meme.
</Tip>

### Ownership type

- **Creator trend** — optional owner link for verification.
- **Community trend** — token-holder coordinated; token launch required.

See [Creator vs community trends](/user-guides/verification/creator-vs-community-trends).

### Category

Categories organize Explore around types of internet culture. Marketing copy often lists **Character**, **Meme**, **Trend**, **Game**, and **Other**—if several fit, pick the strongest match. The **live create form** is authoritative; picker labels can ship ahead of static docs.

### Trend socials

Website, **X**, **TikTok**, **Instagram**, **YouTube** when exposed in the form.

## Token fields

### Token image, name, ticker

**Ticker** is shown with **`$`** prefix in feeds and search.

### Token socials

Adds **Telegram** in addition to the trend social list.

## Copy trend detail to token

When launching the first token with a new trend, you can copy image/description/socials as a starting point—still customize when the market needs different positioning.

## Anti-snipe

Optional at launch: limits how much supply any single wallet can scoop in the first minutes after launch. Product copy describes **up to ~2% of supply per wallet** for the **first ~5 minutes**—designed to reduce early concentration from snipers. **Always read the create screen**; caps are enforced in the contract/UI you confirm against.

Trades that would violate limits surface errors such as **Anti-snipe limit exceeded**.

## Creator buy

Initial purchase is denominated in **`IP`**. The UI typically allows choosing an opening buy in the **~0.1%–3% of supply** range (subject to limits shown live), with estimates for tokens received and fees.
