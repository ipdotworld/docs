## Before you create

<Warning>
  If a **very similar trend** already exists, do **not** create another umbrella. **Launch a token** under the existing trend instead: [Launch a token](/user-guides/launch/create-a-token).
</Warning>

The app warns about duplicates during registration; moderators may later merge duplicates.

![Create trend figure](/images/create-trend.svg)

## Ownership types

- **Creator trend** — tied to a clear owner (person, brand, project). Owner link (X or TikTok) helps verification. Launching a token is **optional**.
- **Community trend** — bottoms-up movement. Launching a token is **required** at creation time.

See [Creator vs community trends](/user-guides/verification/creator-vs-community-trends).

## Steps

<Steps>
  <Step title="Open Create trend">
    In the app, go to **Create → Create trend** (wording may show legacy **IP** in UI until fully renamed—same flow).
  </Step>
  <Step title="Identity">
    Add **image**, **canonical name**, and **aliases** so search catches real spellings.
  </Step>
  <Step title="Pick ownership type">
    **Creator** vs **Community** controls whether token launch is optional.
  </Step>
  <Step title="Category, description, socials">
    Add enough context that traders are not guessing.
  </Step>
  <Step title="Launch token (optional vs required)">
    **Community** forces token launch on. **Creator** can toggle.
  </Step>
  <Step title="Confirm">
    Submit and complete wallet prompts when launching a token in the same flow.
  </Step>
</Steps>

## Launch fees

Token launches charge **`IP`** as shown in the create UI (commonly **10 IP** for spam deterrence—confirm the exact number in-product before you confirm a transaction).
