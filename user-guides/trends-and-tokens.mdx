## Trends

A **trend** is the canonical home for an idea: a meme, a creator storyline, a brand moment, or a community movement. One trend can have **many tokens** attached—different launches or “remixes” of the same underlying theme.

Trends exist so the platform can:

- group markets that belong together,
- route fees and rewards consistently,
- power discovery (Explore) without everyone fighting duplicate pages.

![Diagram: trends group multiple tokens](/images/explore-default.svg)

## Tokens

A **token** is a **single tradable market** under a trend. This is where price, liquidity, graduation, charts, and the trade panel live.

If the app UI still labels something **Remix**, read that as **token**—same object, older label.

## Why both exist

If you are a **degen**: think “parent folder + shitcoins inside.”

If you are a **normie**: think “topic page + separate tickers for different launches around that topic.”

<Tip>
  When you **create**, you either spin up a **new umbrella** for an idea or **launch a token** under an umbrella that already exists (Gitbook wording: link an existing idea **or** create a new one first).
</Tip>
