## Layout (Gitbook / product)

Many builds use a **two-panel** layout:

**Left / summary column**

- Profile identity and **rank**,
- **Connected socials**,
- Key stats (**points**, **tokens**, **referrals**, **posts**),
- **Referral link** (username-based URL, one-click copy),
- **Unrealized PnL** where enabled.

**Main column**

- **Holdings** (created tokens, owned tokens, market data + PnL),
- **Activity**,
- **Feed** (your indexed **X** / **TikTok** with engagement, earnings, and deep links to posts).

## Header

Your profile typically includes:

- **Username** (set early—referral links depend on a stable public handle),
- **Bio** and **avatar**,
- **Wallet address** (truncated in UI lists),
- **Linked socials** (icons may respect visibility toggles if enabled),
- **Balances and volume** summaries where the app exposes them.

## Ranking tiers

Leaderboard / profile **ranks** are described in product copy as five tiers:

**Normie → Casual → Degen → Trencher → Legend**

Points accrue from **trading**, **token creation**, **referrals**, and **engagement** (exact weights: see [Points and leaderboard](/user-guides/rewards/points-and-leaderboard)).

## Tabs

Profile tabs can include **Tokens**, **Trends**, **Takes**, **Referrals**, and other activity views as shipped—use the live app as source of truth for ordering and labels.

## Referrals

Copy your referral link from profile. Format:

`https://ip.world?ref=YOURCODE`

See [Referral program](/user-guides/rewards/referrals) for **fee-share** vs **points** mechanics.

## PnL tracking

Where enabled, surfaces include **per-token PnL**, **total unrealized PnL**, and **shareable PnL cards**.
