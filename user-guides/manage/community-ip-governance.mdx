## The model

Community IPs are coordinated by token holders.

On `ip.world`, community governance is built around:

- a **canonical voting token** (the first token created for that IP)
- proposals + voting powered by Snapshot (the UI links out to Snapshot for details)

## Access: token-holder gated

To access governance actions (like creating proposals), the app checks whether your wallet holds a balance of the canonical voting token.

## What you can do

Typical governance actions include:

- **vote** on proposals (For / Against / Abstain)
- **create proposals** (when eligible)

There are also spending-style proposals used for treasury operations (for example, proposing to claim vested tokens/fees to the community treasury).

## How proposals pass (voting rules)

In plain English: proposals need **more "For" than "Against"** _and_ they need enough "For" votes to clear a baseline + opposition threshold.

The app uses these pass conditions:

- **YES > NO**
- **YES ≥ REQUIRED_YES**

Where:

- **REQUIRED_YES = B + k × NO**
- **B = 10%** (baseline)
- **k = 1**

<Tip>
  If a proposal is controversial (high "Against"), it needs meaningfully more "For" to pass — even if it would normally clear a simple percentage quorum.
</Tip>

## Practical tips

<Tip>
  If voting is "shielded" on Snapshot, you may see the UI indicate shielded voting and you won't see your explicit choice in-app.
</Tip>
