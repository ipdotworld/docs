## IP fields

### IP Image

Your IP’s primary visual identity. This is what users see in lists and cards.

### IP Name

The canonical name for the IP.

### IP Alias

Aliases are additional identifiers (nicknames, alternate spellings, common search terms).

<Tip>
  Use aliases to capture how people actually search for the meme/creator.
</Tip>

### Ownership Type

- **Creator IP**: can optionally include an owner link (X or TikTok) to help verification.
- **Community IP**: token-holder owned; token launch is required.

See: [Creator vs community IPs](/user-guides/verification/creator-vs-community-ips)

### Category

Used to organize the platform and improve discovery.

### Description

Keep it short, specific, and helpful. “Vibes” are fine, but include enough context to avoid confusion with lookalikes.

### Social links (IP)

IP socials support:

- Website
- X (Twitter)
- TikTok
- Instagram
- YouTube

## Token fields

### Token Image

Token art / avatar.

### Token Name & ticker

- **Token Name**: the readable label.
- **Ticker**: the symbol users will see with a `$` prefix.

### Description

Explain what this token represents under the IP. (It can be a specific meta, era, or angle.)

### Social links (Token)

Token socials support:

- Website
- X (Twitter)
- TikTok
- Instagram
- **Telegram**

## “Copy IP detail to token”

When you create an IP and choose to launch the first token, you can copy the IP’s image/description/socials into the token form as a starting point.

<Note>
  Copying details is a convenience, not a requirement — you can (and often should) customize token identity.
</Note>

## Anti-snipe

When enabled at launch:

> Only allow wallets to hold a maximum of **2% of supply** for **5 minutes** post creation.

This is designed to reduce early post-launch concentration.

You may see UI messages like “Anti-snipe limit exceeded” if a trade would violate those limits.

## Buy amount (launch)

When launching a token you choose a **buy amount** denominated in `IP`.

The UI estimates:

- expected tokens received
- % of total supply
- max purchase limit (displayed in the UI)
