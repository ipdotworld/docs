## Before you create: check for duplicates

<Warning>
  **Don't create a new IP if an extremely similar/duplicate IP already exists.**  
  Create a **token under the existing IP** instead: [Creating a token](/user-guides/create/creating-a-token).
</Warning>

<Note>
  The app warns about duplicates during registration. If an IP is later deemed a duplicate, it can be removed.
</Note>

## Choose an ownership type

When you create an IP you pick one of two types:

- **Creator IP**: tied to a clear owner (individual, brand, project). You can add an owner link (X or TikTok) to help verification. Launching a token is optional.
- **Community IP**: bottoms-up trend/movement. Community IPs start with a token launch and are verified through metrics like market cap and holder count. Launching a token is required.

See: [Creator vs community IPs](/user-guides/verification/creator-vs-community-ips)

## Create flow

<Steps>
  <Step title="Open the Create IP flow">
    In the app, go to Create → **Create IP**.
  </Step>
  <Step title="Fill out the IP identity">
    - **Image**
    - **IP Name**
    - **IP Alias** (you can add multiple aliases)
  </Step>
  <Step title="Pick ownership type">
    Choose **Creator** or **Community**.
  </Step>
  <Step title="Add category, description, socials">
    Add enough context so users can tell what this IP is.
  </Step>
  <Step title="Launch token (optional for creator, required for community)">
    - Creator IP: you can toggle "Launch Token" on/off  
    - Community IP: Launch Token is forced on
  </Step>
  <Step title="Create">
    Submit the form and follow wallet prompts if you're launching a token.
  </Step>
</Steps>
