## The core relationship

- **An IP is the umbrella.**
- **Tokens are created under an IP** (you’ll see them listed on the IP page).

When you’re deciding what to create, use this rule of thumb:

<Tip>
  **If it’s the same “thing” (same meme/creator/brand/trend), don’t create a new IP.**  
  Create a **token under the existing IP** instead.
</Tip>

## Why not duplicate IPs?

Duplicating an IP splits the identity:

- Users won’t know which page is “real.”
- Verification and ownership signals become messy.
- Liquidity and attention fragment across near-identical pages.

If an IP exists and it’s close-enough, build under it.

## What “tokens under an IP” represent

Tokens under the same IP can represent different angles, cultures, or meta-narratives around the same identity.

On token pages you’ll often see signals like:

- Market info
- “Official-ish” endorsements (when the IP owner is verified)  
  See: [Endorsements](/user-guides/manage/endorsements).
