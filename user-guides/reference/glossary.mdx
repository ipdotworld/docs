title: "Glossary"
description: "Definitions for the ip.world ecosystem"
---

## IP

The umbrella identity page (meme/creator/brand/trend). Tokens can be created under an IP.

## Token

A market/remix created under an IP.

## Remix

In the app UI/code, tokens are often referred to as "remixes" under an IP.

## Creator IP

An IP type tied to a clear owner. Can be claimed by the owner after approval.

## Community IP

An IP type coordinated by token holders. Launching a token is required.

## Endorsement

A signal from a verified IP owner applied to a token (Official / Liked / Flagged).

## Anti-snipe

A launch option that limits wallet concentration immediately after token creation.

## ITL (IP Token License)

The license framework used by `ip.world` for verified IP ownership.

## IPA (IP Account)

A Story "IP Account" address used during claiming/registration flows (advanced claim path).

## Canonical voting token

For community IP governance, the app uses the **first token created under the IP** as the canonical token for token-holder gating.

## Quorum / pass conditions (governance)

When the app decides whether a governance proposal passed:

- **YES must be greater than NO**
- **YES must be at least a baseline plus an opposition-adjusted threshold**

The rule is:

- **REQUIRED_YES = B + k × NO**
- **B = 10%**
- **k = 1**
